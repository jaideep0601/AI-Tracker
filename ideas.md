# AI Tracker - Ideas & Tasks

## Overview
This document tracks ideas, goals, and tasks for the AI Tracker project.

## Current Goals
- [ ] Create a system to track AI developments and stay updated
- [ ] Organize and manage AI learning materials efficiently
- [ ] Goal 3

## Ideas
- **Point 1**: It's taking a lot of time reading and understanding about AI, but often I feel like there are a lot of developments happening in AI which I cannot keep track of
- **Point 2**: The field is evolving so fast with new research papers, new GitHub repositories, new projects, new large language models and other models being released on a weekly basis. It feels like even if you're not keeping track for two weeks, you are falling behind a lot
- **Point 3**: Build a website that serves as a single platform to keep track of all the things that are happening in AI
- **Point 4**: Track relevant AI content from multiple sources: People, Companies, Research Papers, GitHub Repos, X.com, LinkedIn, Reddit, Top Researchers, Conferences, etc.
- **Point 5**: Implement a feedback system where you can continuously give preferences and refine the platform output if the current display is not meeting your needs
- **Point 6**: Based on preferences, the website should show more relevant content. Start with a simple thumbs up/thumbs down system (details to be discussed during development). Single-user website for personal use - no authentication needed.
- **Point 7**: In later stages, add user authentication so individual people can use the website, store their own preferences, and have personalized content displayed based on their preferences
- **Point 8**: Determine data refresh frequency - how often should information be updated? (e.g., once a day, every two days, etc.) This needs to be planned to balance freshness vs. system load
- **Point 9**: Need a compelling project name that reflects the purpose of tracking and monitoring AI developments
- **Point 10**: Create a comprehensive, well-organized requirements document (IDEAS_V2) that systematically consolidates all ideas with clear sections, technical details, and structured information
- **Point 11**: Interface design requirements - must be extremely modern and user-friendly with full responsiveness. Mobile-first approach essential since the website will be frequently accessed on mobile phones while traveling. High mobile optimizations required for seamless experience across all screen sizes.
- **Point 12**: Create a detailed plan for skills and sub-agents needed during website development to ensure proper task execution and specialized expertise for different components
- **Point 13**: Need a source discovery and quality rating system to handle noise from social media platforms (LinkedIn, X.com, Reddit). Users should be able to rate sources (1-5 stars), and only 4+ star rated sources get scraped into the website. This ensures high-quality, relevant content while filtering out noise from cluttered platforms.
- **Point 14**: Comprehensive filtering system required to display and organize content by multiple dimensions - category filters (papers, repos, people, companies, etc.), source filters (GitHub, X, LinkedIn, Reddit), quality filters (recency, popularity, relevance), topic filters, and advanced filters with save/preset capabilities. Users must have granular control over what content appears on their feed.

## Implementation Approach
- **Planning First**: Create detailed roadmaps before implementing
- **Staged Development**: Work in phases/versions with clear milestones
- **Development → Production**: Build and test in development environment first, then deploy to production
- **Version-based Planning**: Each stage should have its own planning document
